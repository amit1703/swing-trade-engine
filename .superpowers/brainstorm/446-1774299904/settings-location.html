Where should Dark Mode & Language live?

Both toggles need to be always accessible. Pick the placement that feels most natural.

A

Settings Page (dedicated)

Both toggles live in the Settings page you already have (currently "coming soon"). You navigate there to change theme or language. Clean separation — settings are in one place.

Pros

  • All preferences in one place
  • Expandable for future settings

Cons

  • Need to navigate away to toggle
B

TopBar (always visible)

Small icon buttons in the top-right of the header — a sun/moon icon for theme and a flag/globe for language. One click from anywhere in the app.

Pros

  • Instant access, no navigation
  • Standard pattern (GitHub, VS Code)

Cons

  • Adds icons to an already-busy TopBar
C

Both — Settings Page + TopBar shortcuts

Settings page is the "home" for preferences. TopBar also has the sun/moon + language icons as quick shortcuts. Clicking in TopBar changes it instantly; Settings page shows the current state.

Pros

  • Best of both worlds
  • Settings page feels "complete"

Cons

  • More UI surface to build
